GitHub Pages is a free and easy way to publish a personal site, project page, or blog using static HTML and themes. No server setup or backend is needed.

For this site, I used the Minimal theme — a clean, lightweight Jekyll theme ideal for blogs and portfolios.

Here’s a simple way to get started:

  1. Create a GitHub account (if you don’t already have one): github.com/join
  2. Create a new repository named yourusername.github.io — this name is important as it tells GitHub to treat it as a website.
  3. Enable GitHub Pages in the repo settings:
    • Go to Settings → Pages
    • Under “Source”, choose the branch (usually main) and folder (typically /root or /docs)
  4. Add a theme:
    • Create a file called _config.yml in the root of your repo
    • Add this line to use the Minimal theme:
      theme: jekyll-theme-minimal
  5. Add an index.md or index.html file with your content
  6. Commit and push your changes — your site should be live at https://yourusername.github.io

For full instructions, see: GitHub Pages: Getting Started
