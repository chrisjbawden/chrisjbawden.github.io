Good‑bye heavy Portainer, hello lightweight DockerManager!

I’ve been running Cockpit for a while now, mostly because its web console lets me wrangle my home‑lab servers from the couch. Originally I bolted on an iframe app that simply embedded Portainer so I could at least monitor containers without leaving Cockpit. It worked, but Portainer’s footprint always felt a bit over‑the‑top for the couple of clicks I actually needed. Also, I have always wanted to contribute to a big project.

Over a quiet weekend I decided to write a native Cockpit application instead of another embed. The result is DockerManager—a single‑page Cockpit plugin that surfaces just the essentials:

  • Container list with start / stop / restart actions and port information (with clickable links)

Why it’s faster & lighter

DockerManager talks straight to the local Docker socket using Cockpit’s channel.open API—no extra daemons, no API proxies. The whole UI is just vanilla JS, PatternFly 4 for styling, and one manifest.json so Cockpit registers it.

So bundle size = small and launch time = short.
